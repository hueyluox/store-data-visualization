# Git 快速同步指南

## 🚀 一键自动推送

### 方法1：使用自动脚本（推荐）
```bash
cd "/Users/oscar/Downloads/门店数据可视化系统"
./git-auto-push.sh
```

### 方法2：手动命令
```bash
cd "/Users/oscar/Downloads/门店数据可视化系统"
git add .
git commit -m "更新文件"
git push
```

## 📋 脚本功能说明

`git-auto-push.sh` 脚本会自动：

1. **检测更改** - 检查是否有文件被修改
2. **显示状态** - 显示文件更改状态
3. **添加文件** - 自动添加所有更改到暂存区
4. **智能提交** - 使用带时间戳的提交信息
5. **推送代码** - 自动推送到GitHub仓库
6. **状态反馈** - 显示操作结果和最新提交信息

## 💡 使用建议

### 何时使用？
- 修改了HTML、CSS、JavaScript文件后
- 更新了数据文件或文档

- 重要节点需要备份时

### 提交信息规范
自动脚本会生成标准格式的提交信息：
```
自动更新 2025-11-10 15:55:30

📊 更新内容:
- 门店数据可视化系统文件修改
- 自动同步到GitHub仓库
```

### 最佳实践
1. **定期同步** - 完成重要修改后立即推送
2. **保持整洁** - 推送前确认文件修改内容
3. **查看结果** - 确认推送成功后再继续工作

## 🔧 故障排除

### 认证问题
如果遇到认证失败：
```bash
# 重新配置token
git remote set-url origin https://hueyluox:YOUR_TOKEN@github.com/hueyluox/-les-bb.git
```

### 网络问题
检查网络连接和GitHub访问：
```bash
ping github.com
```

### 文件冲突
如果遇到文件冲突：
```bash
git pull origin master
# 解决冲突后再推送
```

## 📱 快捷方式

可以创建系统别名或桌面快捷方式：
```bash
# 添加到 ~/.zshrc 或 ~/.bash_profile
alias push-store="cd /Users/oscar/Downloads/门店数据可视化系统 && ./git-auto-push.sh"
```

然后使用：
```bash
push-store
```
